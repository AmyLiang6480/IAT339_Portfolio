Spin City Records

Company Website Design and Development

Collaboration

To collaborate effectively on this project, we used GitKraken, which allowed us to distribute the work and merge it later. We created branches for each feature, which allowed us to work on different aspects of the project simultaneously without interfering with each other's progress.

We also set up a weekly check-in to discuss our progress and ensure that we were on track to meet our deadlines.

Design Process

Our team started the design process by consolidating our style guides into one single document. We ensured that the document included all the materials required and was in the form of a clear, nicely organized style guide.

We built in responsive aspects from the start, ensuring that the website was accessible on all devices. We also designed consistent interfaces and interaction elements to improve user experience.

Code/Development

Our team focused on code validity, readability, and cleanliness of folder structure. We ensured that our code was consistent with the project requirements, and we used meaningful names for our files and folders.

We followed the best practices for coding to ensure that our code was maintainable and easy to read. We also tested our code to ensure that it was working correctly and fixed any issues we encountered promptly.

Content/UX

Our team focused on effective information architecture, relevant content, and responsive and usability. We conducted user testing from peers to gather feedback on the website's usability, making necessary changes to improve user experience.

We ensured that our website was responsive for any device. We also made sure that the content was informative, engaging, and relevant to our target audience.

Challenge

Since the team consisted of three members who worked on different devices(desktops), there were times when conflicts and errors in layout, color, and appearance arose. This challenge threatened to undermine the website's overall user experience and accessibility.

Solution

We used tools such as media queries and flexible grid layouts to ensure that the website looked and functioned correctly across different devices.

By prioritizing responsive design and rigorous testing, we team successfully overcame the challenge of ensuring that the website looked and functioned correctly across all devices.

Conclusion

We created a user-friendly, accessible, and responsive company website. The project involved collaboration using GitKraken, distributing the work, and merging it later.

We followed a design process that prioritized effective information architecture, relevant content, and website accessibility and usability.

We also focused on code validity, readability, and cleanliness of folder structure. Overall, this project provided us with valuable experience in collaborating effectively as a team, building a website, and prioritizing user experience.
